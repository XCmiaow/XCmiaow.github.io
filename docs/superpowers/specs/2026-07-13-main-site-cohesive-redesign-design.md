# 主站协调性重构设计说明

日期：2026-07-13  
状态：已确认，待实施

## 目标

把“硅基余烬”主站从偏海报化、装饰信息过多的长页面，重构为更克制、协调、可阅读的编辑型主页；同时修复黑洞越界和字体回退问题。个人简历保留全部真实履历，只删除重复能力说明、重复标签和不必要的二次解释。

## 已确认的问题

- 1440px 视口下，黑洞场景右边界达到 1515px；390px 视口下达到 619px。
- `EmberField.astro` 使用负向右边距和偏右的 `--hole-x`，视觉主体与内部绘制范围都被推到视口之外。
- 页面声明了未实际加载的 `Satoshi` 与 `Source Han Serif SC`，最终依赖不稳定的系统回退字体。
- 首页同时存在导航、首屏按钮和“主站入口”三套入口，信息重复。
- `/profile` 在 390px 视口下约 8931px 高，包含 27 个内容块；项目的“能力”、标签和说明互相重复，技能与研究方向也重复解释。

## 方向选择

采用 A：克制的编辑型主站。

未采用的方向：

- 全屏居中黑洞：冲击力强，但会持续影响正文对比度。
- 将黑洞缩成品牌图标：最安全，但会削弱“硅基余烬”的核心识别。

## 主站信息结构

首页调整为以下顺序：

1. 品牌首屏：品牌名、两句定位、两个主要动作和黑洞。
2. 核心理念：一句主张和三条简短原则。
3. 最新文章：保留三篇，使用紧凑列表。
4. 方法：四步流程压缩为一条连续路径。
5. 资源：课程、任务说明、Prompt 与验收清单。

删除独立的 `SiteCompass` 首页入口区。顶部导航与首屏动作已经承担入口职责，不再重复。

## 首屏与黑洞

### 布局

- 首屏仍使用绝对定位的视觉层，但 `.ember-stage` 必须完整约束在 `.ember-hero` 内。
- 禁止使用负向 `inset`、负向右边距或超过视口的舞台宽度。
- 桌面端黑洞中心位于首屏宽度约 64%–66% 处，尺寸上限约 480px。
- 手机端黑洞中心位于首屏宽度约 62%–64% 处，尺寸不超过首屏宽度的 70%。
- 舞台使用 `overflow: clip`；黑洞主体、光子环与吸积盘在视觉上完整，不再只露出右侧局部。

### 层次

保留远景尘埃、后方吸积盘、事件视界、前方吸积盘和近景颗粒五层结构，但减少轨道线、观测参数和伪仪表文字。层次依靠亮度、模糊、透明度和运动速度区分，不依靠更多标签。

### 首屏文字

- 主标题最大字号从 7rem 降至约 5.2rem。
- 删除 `OBS.01 / ARCHIVE ACTIVE` 等三段式观测栏，仅保留一个小型品牌状态行。
- 首屏只保留两个主要动作：博客与资源；课程入口放入资源区。
- 文案宽度限制在可读范围内，不与黑洞主体重叠。

## 字体系统

不依赖未加载字体，也不增加远程字体请求。

- 中文正文与标题：`PingFang SC`、`Hiragino Sans GB`、`Noto Sans CJK SC`、`Microsoft YaHei`、系统无衬线。
- 英文与数字：`Avenir Next`、`Segoe UI Variable`、`Segoe UI`、系统无衬线。
- 元数据：`ui-monospace`、`SFMono-Regular`、`Consolas`。
- 标题与正文使用同一无衬线家族，通过字号、字重、字距和行高建立层级，避免宋体回退造成的突兀海报感。

## 内容区视觉规则

- 主内容最大宽度保持在 1200–1280px，不再让各组件自行扩大画布。
- 区块标题最大约 3.4rem，正文最大 65 个字符宽。
- 减少等宽三栏和巨型标题，使用统一基线、细分隔线与少量错位建立节奏。
- 方法区桌面为连续水平路径，手机为单列时间线。
- 最新文章与资源均使用列表，不使用同尺寸卡片墙。
- 动效只使用 `transform` 与 `opacity`；继续支持 `prefers-reduced-motion`。

## 组件边界

- `SiliconAshesHome.astro`：只负责首页区块顺序与数据传递。
- `EmberHero.astro`：负责首屏文字与视觉层的组合。
- `EmberField.astro`：只负责受约束的黑洞视觉，不决定页面宽度。
- `BrandManifesto.astro`：只负责核心理念与三条原则。
- 新建 `MethodStrip.astro`：负责四步方法路径，使方法区可以独立排在文章之后。
- `WritingPreview.astro` 与 `SignalLinksSection.astro`：保留职责，但统一更紧凑的列表节奏。
- `silicon-embers.css`：提供真实可用的字体栈、容器、字号和间距令牌。

## 个人简历精简

保留教育、全部奖项、全部项目、技能、学生经历、志愿服务、研究方向和公开联系方式，不删除事实条目。

删除或合并：

- 奖项卡片中的“国际级竞赛”“能力证明”等重复摘要。
- 技能分类下的概述句，只保留分类名与技能清单。
- 项目中的“能力：”段落和标签行，只保留项目名与一段事实描述。
- 研究方向卡片中的逐项解释，只保留总体研究说明和方向名称。
- 页面底部重复联系方式；联系方式保留在首屏资料区。
- 一页、学术和求职简历中的项目“能力”段落，保留项目事实描述。

底层履历数据不删除，精简发生在展示层，避免影响证明材料和其他数据消费者。

## 无障碍与异常状态

- 所有导航与动作维持可见焦点、44px 触控目标和语义化标题层级。
- 黑洞继续标记为装饰性内容，不进入无障碍树。
- 无 JavaScript 时页面内容与黑洞静态层仍可见。
- 画布初始化失败时不影响 CSS 黑洞和正文。
- 修复个人简历页缺少 favicon 引起的浏览器 404。

## 验收标准

- 360、390、768、1024、1440px 五组视口中，页面 `scrollWidth` 不超过 `innerWidth`。
- 黑洞主体位于首屏内部，手机与桌面都不再被右侧裁掉。
- 源码中不再出现黑洞舞台的负向右侧扩张规则。
- 全站不再声明未加载的 `Satoshi` 或 `Source Han Serif SC`。
- 首页不再渲染独立 `SiteCompass` 区。
- `/profile` 与三种简历页面不再渲染“能力：/Capabilities:”项目段落。
- 奖项、技能、项目和研究方向的事实条目数量保持不变。
- 深色、浅色、移动端、键盘导航和减少动态效果模式均通过检查。
- `npm run check` 完整通过。

## 非目标

- 不重写博客、课程和资源页的内容。
- 不删除履历数据源中的事实记录。
- 不更换 Astro 架构或引入前端框架。
- 不新增远程字体、图片或第三方运行时依赖。
