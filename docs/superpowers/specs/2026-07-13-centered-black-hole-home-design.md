# 居中黑洞双幕式首页设计说明

日期：2026-07-13  
状态：已确认，待实施

## 目标

把“硅基余烬”首页收束为两个连续画面：以居中黑洞为唯一视觉主体的首屏，以及承担公开联系入口的底部联系页。顶部导航继续提供博客、资源、关于和语言切换入口；首页不再重复展示文章、方法、资源或其他介绍内容。

## 已确认方向

采用 A：双幕展馆式首页。

- 第一幕是完整首屏，黑洞位于画面中心，品牌信息分置左上和右下。
- 第二幕是联系页，集中展示邮箱、企业微信、GitHub、个人信息和公众号入口。
- 不采用“首屏 + 极简小页脚”，因为联系区域的存在感不足。
- 不采用“联系信息叠在首屏底部”，因为会破坏黑洞周围的留白和焦点。

## 首页信息结构

首页只保留以下结构：

1. 顶部品牌导航。
2. 居中黑洞首屏。
3. 底部联系页。

首页停止渲染以下组件：

- `BrandManifesto`
- `WritingPreview`
- `MethodStrip`
- `SignalLinksSection`

这些组件和对应博客、资源、课程、关于页面不删除。用户仍可通过顶部导航进入独立页面。

## 首屏构图

### 黑洞

- 黑洞中心在首屏容器的水平与垂直 50% 位置，不再因视口宽度向右偏移。
- 桌面端场景尺寸使用约 `min(42vw, 560px)`，相比当前 480px 上限增大约 17%。
- 手机端场景尺寸使用约 `min(78vw, 320px)`，同时缩短吸积盘横向比例，保证光盘主体不因放大而明显裁切。
- `.ember-stage` 继续完整约束在 `.ember-hero` 内并使用 `overflow: clip`，任何视口都不产生横向滚动。
- 保留远景尘埃、引力薄雾、后方吸积盘、光子环、事件视界、前方吸积盘和近景颗粒的现有分层。

### 事件视界层次

事件视界不使用单一纯黑色块。中心使用暖黑到深黑的径向渐变，并增加三种细节：

- 极弱的偏暖内部反光，让中心黑色与页面背景产生色相差。
- 内侧暗角，保持事件视界的深度和吞噬感。
- 一圈低亮度光子边界，确保轮廓在深色背景下仍可辨认。

这种差异必须克制：中心仍然首先被识别为黑洞，而不是发光球体。

### 文字象限

左上信息组包含：

- 小型状态行。
- `silicon-embers` 英文标识。
- 中文或英文主标题。
- 主标语。

右下信息组包含：

- 副标语。
- 一行小型观测元数据，作为视觉收尾。

首屏删除主按钮、次按钮和滚动提示。博客、资源与关于入口由顶部导航承担，避免同一入口重复出现。

### 响应式规则

- 桌面与平板保持左上 / 右下对角构图，文字不得覆盖事件视界主体。
- 手机端仍保持对角关系，但通过纵向空间把左上文字放在黑洞上方、右下文字放在黑洞下方。
- 首屏高度使用动态视口单位，避免移动浏览器地址栏造成跳动。
- 360px 宽度下标题、标语和导航均不得产生横向溢出。

## 底部联系页

现有 `SiliconEmbersFooter` 增加首页专用的 `contact` 变体：

- 首页使用较完整的联系页布局，最小高度约半屏。
- 左侧展示 `contact` 标识、联系标题和品牌说明。
- 右侧以清晰列表展示邮箱、企业微信、GitHub、个人信息与公众号。
- 其他品牌子页面继续使用当前紧凑 Footer，不改变其页面长度。
- 联系链接保持至少 44px 触控高度、键盘焦点样式和外链安全属性。

## 组件边界

- `SiliconAshesHome.astro`：只组合 `EmberHero`，不再读取博客 Content Collection。
- `EmberHero.astro`：负责两组首屏文字、语义化标题和黑洞视觉组合。
- `EmberField.astro`：负责严格居中的黑洞尺寸、层次与画布效果。
- `SiliconEmbersSiteFrame.astro`：根据当前是否为品牌首页选择 Footer 变体。
- `SiliconEmbersFooter.astro`：保留默认 Footer，并新增首页联系页样式。
- `siliconEmbersHome.ts`：继续作为中英文文案来源；不删除独立页面仍在使用的数据。

## 无障碍与降级

- 页面仍只有一个 `h1`，黑洞保持 `aria-hidden="true"`。
- 顶部导航、语言切换、主题切换和联系链接保留可见焦点。
- 禁用 JavaScript 时 CSS 黑洞、品牌文字与联系入口仍然可见。
- `prefers-reduced-motion` 下停止黑洞循环动画，构图保持不变。
- 深浅主题均应能分辨事件视界轮廓。

## 验收标准

- 首页主内容只渲染一个首屏 section；其后直接进入首页联系 Footer。
- 首页源码不再导入或渲染 `BrandManifesto`、`WritingPreview`、`MethodStrip`、`SignalLinksSection`。
- 首屏不再渲染 `.hero-actions` 或 `.scroll-cue`。
- 360、390、768、1024、1440px 五组视口中，黑洞场景中心与首屏中心的水平、垂直误差均不超过 2px。
- 1440px 下黑洞场景宽度不小于 540px；390px 下不小于 285px。
- 所有验收视口中 `scrollWidth` 不超过 `innerWidth`。
- 深色主题下事件视界中心和页面背景的计算色值不同，轮廓可见。
- 首页联系 Footer 存在且邮箱、GitHub、个人信息等现有链接有效。
- 其他博客、资源、课程、关于和简历路由继续正常构建。
- `npm run check` 完整通过。

## 非目标

- 不删除博客、资源、课程、关于或简历页面。
- 不修改课程内容和履历数据。
- 不引入新图片、字体、前端框架或第三方运行时。
- 不改变顶部导航的信息架构。
