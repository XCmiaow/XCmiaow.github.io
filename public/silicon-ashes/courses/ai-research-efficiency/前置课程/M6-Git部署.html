M6 · Git 极简 + Pages 部署

5 分钟理解版本控制 + 10 分钟获得第一个公开网站

时长
约 15 分钟
成果
一个全世界可访问的个人学术页面

第一部分：Git 在 5 分钟内——只讲概念

时长
约 5 分钟
关键词
版本控制、init、add、commit、push

Git 是什么？

Git = Word 修订模式，但聪明 100 倍
你在 Word 里写论文，打开修订模式。每次修改都有一个记录：谁改的、什么时候改的、改了什么。你可以随时回退到昨天的版本。

Git 就是全自动的修订模式——你的整个项目文件夹。

写代码、写论文、做数据分析——每一次保存（commit）都有记录。三个月前的版本随时可以恢复。和同事协作时，谁改了哪一行一清二楚。

区别在于：Word 修订模式的记录是一次性的（关闭就没了），Git 的记录是永久的、可以跳转到任意历史节点的。

GitHub 是什么？

GitHub = 网盘 + 社交网络，只是里面装的是代码（和 HTML 网页）
网盘功能：把你的项目上传到云上，换电脑也能继续工作，丢了电脑代码还在。
社交功能：别人可以看到你的项目、给你提建议（Issue）、帮你改代码（Pull Request）。

更重要的是：GitHub Pages 可以免费把你的 HTML 文件变成一个真正的网站。Web 地址是 你的用户名.github.io/项目名——全世界任何人都能访问。

全部只学四个词

命令翻译什么时候用生活类比
git init 初始化：在这个文件夹启用版本控制 创建一个新项目的第一步 买一本新的工作笔记，在第一页写上"XX 项目工作日志"
git add . 暂存：把修改加入"待保存清单" 改完文件之后、存档之前 把要打印的文件放在打印机的"待打印"队列里
git commit -m "做了什么" 提交：正式存档，附带说明 每次完成一个可描述的修改 在笔记本上写下一行"7月3日：完成了文献综述大纲"
git push 推送：把本地存档同步到 GitHub 云端 想让别人看到你的更新时 把笔记本复印一份放到档案室（云端备份）
Git 工作流的完整循环：

改文件（你在本地编辑） add（告诉 Git "我要保存这些修改"） commit（正式存档，写一句说明） push（同步到 GitHub 云端）

重复这个循环，每次 commit 都是项目的一个快照。三个月后，你可以回到任何一个 commit 的版本。
明确声明：不讲分支、合并、冲突

Git 是一个强大的工具，但本课只需要你掌握上面四个操作。分支（branch）、合并（merge）、冲突解决（conflict resolution）是 Git 的高级功能——这些留给有需要的同学课后自学。

你现在需要的 Git 知识，就是这 4 个词 + 1 个循环。足够你在接下来的 WorkBuddy 课程中正常使用。

第二部分：Pages 部署——10 分钟获得你的第一个网站

时长
约 10 分钟
成果
一个全世界可访问的 URL

下面每一步请跟着操作。最终你会得到一个网址，用手机扫二维码就能访问你的个人学术页面。

第一步：创建 GitHub 仓库

1

创建 GitHub 账号并新建仓库（约 2 分钟）

  1. 打开 github.com，注册账号（如果有账号，直接登录）
  2. 登录后，点击右上角的 "+"号 New repository
  3. Repository name 填写：my-lab-page
  4. Description 填写：My academic lab introduction page
  5. 选择 Public（公开仓库——免费且符合 Pages 要求）
  6. 勾选 "Add a README file"
  7. 点击 "Create repository"

第二步：上传 HTML 网页模板

2

在仓库中创建你的个人实验室介绍页面（约 2 分钟）

  1. 在仓库页面，点击 "Add file" "Create new file"
  2. 文件名填写：index.html
  3. 将老师提供的实验室介绍 HTML 模板粘贴到编辑区（或你自行编写一个简单的页面）
  4. 修改模板中的内容：你的名字、研究方向、联系方式
  5. 页面底部 "Commit new file" 区域，填写 commit message："创建个人实验室介绍页面"
  6. 点击 "Commit new file"
模板内容参考——你的 index.html 至少应该包含：
- 你的姓名和职称
- 研究方向（一句话）
- 所在学校/学院
- 联系邮箱
- 一张个人/实验室照片（可选）

不要担心页面简单。一个只有姓名和研究方向的页面，已经比 95% 的化学老师在网上有更强的可见度了。后续 WorkBuddy 课程会大幅丰富它。

第三步：启用 GitHub Pages

3

让仓库变成一个真正的网站（约 1 分钟）

  1. 在仓库页面，点击顶部导航的 "Settings"
  2. 在左侧菜单找到并点击 "Pages"（在 "Code and automation" 分组下）
  3. "Source" 下拉菜单选择 "Deploy from a branch"
  4. "Branch" 下拉菜单选择 "main"，旁边选择 "/ (root)"
  5. 点击 "Save"

第四步：等待部署，访问你的网站

4

等待 30-60 秒，刷新页面（约 2 分钟）

  1. 保存后，Pages 设置页面会显示一条信息："Your site is being built..."
  2. 等待约 30-60 秒，刷新页面
  3. 当显示 "Your site is live at" 时，你的网站已经部署完成
  4. 网址格式：https://你的用户名.github.io/my-lab-page/
  5. 点击链接——这就是你的个人学术网站！

第五步：用手机扫码——见证"我的页面在互联网上了!"

5

关键瞬间（约 2 分钟）

  1. 打开浏览器，访问 https://你的用户名.github.io/my-lab-page/
  2. 在地址栏左侧会看到一个"二维码"图标（或用浏览器自带的分享功能生成二维码）
  3. 用手机扫描二维码
  4. 手机会自动打开你的个人页面

这一刻，请停下来感受一下：

你刚刚把一段 HTML 代码变成了一个全世界任何人都能访问的真实网站。

你在实验室的电脑前改了一行字 commit push 30 秒后，地球另一端的某个人打开你的页面就能看到更新。

这不是程序员才有的能力。这是你，一个化学/化工研究者，在前置课程中就能做到的事。
