术语速查

英文黑话与课堂解释

术语课堂解释
LLM大语言模型。它根据上下文预测下一个 token，不是一本不会错的百科全书。
Token模型眼里的文字单位。中文一个字、英文单词片段、标点都可能算 token。费用和上下文窗口都和它有关。
上下文窗口模型一次能看到的材料范围。超过范围后，旧内容会被压缩或遗忘。
幻觉模型给出看似合理但没有证据的内容。解决方法不是“相信更贵模型”，而是给来源、跑验证。
API软件之间约定好的接口。像餐厅点餐口：你按格式点菜，后厨按规则返回结果。
API Key调用 API 的钥匙。它等于能花你额度的门禁卡，不能截图、不能提交到 GitHub。
Prompt给模型的任务单，不是咒语。好 prompt 说清目标、材料、限制、输出形式、验收标准。
Coding PlanAI 动手前的设计图。像装修前先看图纸，避免边拆边想。
Context Engineering上下文工程。不是写一句神奇提示词，而是把模型需要看的文件、规则、例子、错误信息组织好。
Harness把模型包起来的工作装置：指令、工具、文件、权限、测试、日志、人工确认。像实验台，而不只是口头命令。
LoopAgent 的循环：计划 -> 修改 -> 运行 -> 观察 -> 修复 -> 记录。循环质量决定任务能不能稳定完成。
Agent会使用工具并持续观察结果的 AI 助手。聊天模型只说，Agent 能读文件、改文件、跑命令。
Server / 服务器远处一直开着的电脑。你打开网页、调用 API、访问模型，本质上常常是在和远处服务器说话。
中转站 / Proxy / Relay请求先到一个驿站，再被转给真正服务。它可能负责换路线、换格式、统一入口或隐藏复杂配置。
RAG检索增强生成。先从资料库找相关材料，再让模型基于材料回答。适合问自己的文档。
MCP模型上下文协议。让 AI 工具以统一方式连接外部工具和数据源。初学者先知道概念即可。
CLI命令行界面。用文字发指令。AI 编程工具喜欢 CLI，因为命令可复制、可记录、可自动执行。
GUI图形界面。用按钮、菜单、窗口操作。对人友好，但不如 CLI 容易被 Agent 稳定复现。
