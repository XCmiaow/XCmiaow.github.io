Day 2 下午核心材料

工作流工坊：把 WorkBuddy 变成科研流程

最后一讲不强制做网页，而是让每位老师用自己的真实科研任务，设计一套可复用、可检查、可迭代的个人科研 AI 工作流。

WorkBuddy 主力场景

科研效率部分以 WorkBuddy 为中心

这部分不再把 Codex 当主角。Codex 是工程化平替，WorkBuddy 才是大多数老师日常科研任务的主入口。

文献阅读与综述

用 WorkBuddy 汇总论文、生成文献矩阵、提出争议点和后续阅读路线。

论文写作与润色

把提纲、实验结果、目标期刊要求交给 WorkBuddy，让它生成草稿和修改建议。

实验方案讨论

让 WorkBuddy 扮演审稿人、导师或安全员，检查方案假设、变量和风险。

数据整理与可视化

先让 WorkBuddy 解释字段、设计图表和检查异常，再决定是否交给代码工具执行。

课程资料生成

生成课堂案例、练习题、作业说明和评分标准，再由老师把关。

项目申请与汇报

打磨研究问题、创新点、技术路线、风险应对和汇报结构。

工作流选题

允许完全开放，但必须落到工作流

最终成果不是“随便做个东西”，而是把一个真实科研任务拆成输入、工具处理、AI 输出、人工检查、保存和复用。

文献综述工作流

输入论文、检索式和研究问题，输出文献矩阵、主题脉络、争议点和下一步阅读清单。

论文写作工作流

输入提纲、实验结果和目标期刊风格，输出段落草稿、审稿人视角问题和修改清单。

数据整理工作流

输入表格、字段说明和目标图表，输出清洗建议、可视化草案和异常值检查表。

课程资料工作流

输入教学目标和已有讲义，输出课程大纲、案例、课堂练习和作业说明。

项目申请工作流

输入指南、研究基础和初步想法，输出结构化提纲、创新点打磨和风险清单。

科研工作流 Loop

把 WorkBuddy 当助理，也把它放进验收流程

AI 工作流不要一口气跑到底。每一轮都要有输入、输出、人工检查和保存位置。

Input准备论文、数据、指南、提纲或课程目标。
Plan让 WorkBuddy 先拆任务和确认输出结构。
Produce生成综述、段落、图表建议、练习题或申请书提纲。
Review老师检查事实、逻辑、引用、隐私和适用性。
Reuse保存 prompt、材料清单、结果和下一轮改进。

WorkBuddy 工作流提示词模板

你是我的科研 AI 助理。请先不要直接生成最终稿，先帮我设计工作流。

科研任务：[文献综述 / 论文写作 / 数据整理 / 课程资料 / 项目申请]
目标用户：[我自己 / 课题组 / 学生 / 评审专家]
输入材料：[论文、数据、指南、提纲、课程目标等]
限制条件：[不能泄露隐私、必须标注不确定结论、需要保留原始来源]
验收标准：
1. 先给 3-5 步工作流，不直接写最终稿。
2. 标出每一步需要我提供什么材料。
3. 标出哪些输出必须人工检查。
4. 给出本次任务可复用的 prompt 模板。

请先输出工作流图和检查表。

人工验收提示词模板

请扮演严谨的同行评审，检查下面这份 AI 输出。

任务背景：[说明这份输出要用于哪里]
输入材料：[粘贴原始材料或摘要]
AI 输出：[粘贴结果]

请检查：
1. 哪些结论缺少依据。
2. 哪些地方可能过度推断。
3. 哪些术语、数据、引用需要我回原文核对。
4. 给出最小修改建议，不要重写整篇。
最终交付

工作流包比单次输出更重要

结课时，每位老师至少带走下面这些材料；GitHub Pages 只作为可选展示页。

个人科研 AI 工作流图

画出输入材料、处理工具、AI 输出、人工检查、保存位置和复用方式。

工具选择表

列出每个科研任务优先用 WorkBuddy、Claude Code、Codex 还是其他工具，并写清理由。

三个个人 prompt 模板

至少覆盖文献、写作、数据/课程/项目申请中的三个场景。

上下文材料清单

列出每类任务要提前准备的论文、数据、要求、示例、格式和禁区。

安全与验收 checklist

检查 API Key、隐私材料、事实来源、人工复核、版本保存和可复用记录。

可选 GitHub Pages 展示页

如果需要公开展示工作流或课程资源，再把结果做成静态网页。

可选展示：GitHub Pages

如果要公开展示，再做网页

小白最容易在仓库名和 Pages 设置上卡住。课堂只在有余力时用投影带一遍。

  1. 新建仓库用户主页用 username.github.io，项目主页用普通仓库名也可以。
  2. 放入入口文件静态网站至少要有 index.html。
  3. 提交并推送本地 Git 或网页上传都可以，课堂优先稳定。
  4. 启用 PagesSettings -> Pages -> Deploy from a branch -> main / root。
  5. 等待发布首次发布可能要等几分钟。打不开时先看 Actions 和 Pages 设置。
