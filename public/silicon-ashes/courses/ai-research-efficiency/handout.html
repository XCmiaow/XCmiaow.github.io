AI 编程与科研效率基础课

给计算机零基础教师的 AI 认知、工具实操与 WorkBuddy 科研工作流教程

浏览器打印时选择“另存为 PDF”；需要 Word 时可从浏览器全选复制到 Word，再微调页眉页脚。

课程目标

不把老师培养成程序员，而是让老师能理解 AI 工具背后的基本逻辑，安全地配置和使用工具，并把 WorkBuddy、Claude Code、Codex 等工具组织成自己的科研效率工作流。

  • 理解 LLM、Agent、API、上下文、提示词、harness、loop 等基础概念。
  • 能在 Windows 或 macOS 上完成 Claude Code、WorkBuddy、Codex 的基础可用性检查。
  • 能设计个人科研 AI 工作流包；GitHub Pages 只作为可选展示方式。

四个半天安排

D1 AM
2 小时

AI 认知与基础概念

让计算机小白先建立地图，不急着装工具，先知道 AI、API、服务器、中转站和终端这些词到底在说什么。

  1. AI 发展史专家系统、搜索引擎、深度学习、ChatGPT、Claude、DeepSeek、Agent 工具。
  2. LLM 基础模型、token、上下文窗口、幻觉、推理模型、费用、速率限制。
  3. 通俗概念API、API Key、Prompt、Coding Plan、服务器、中转站、CLI/GUI。
  4. AI 工具地图聊天助手、搜索增强、科研阅读、WorkBuddy 类工作台、编程 Agent、自动化工具。
课堂产物一张“AI 工具地图”和一张“我能听懂的术语表”。
验收能用生活化语言解释 API、Prompt、Coding Plan、服务器、中转站、CLI/GUI 和幻觉。
D1 PM
2 小时

AI 指挥方法

先学会怎么把任务讲清楚，再去使用具体工具；让老师从“问一句”升级到“设计一套可检查的任务”。

  1. Prompt Engineering目标、材料、限制、输出格式、验收标准。
  2. Context Engineering把资料、文件、示例、错误信息摆上桌，让模型少猜。
  3. Coding Plan / Harness / Loop先计划，再执行；用检查表、权限、日志和人工确认把 AI 放进工作装置。
  4. 科研任务拆解文献综述、论文润色、数据分析、课程资料整理、实验方案讨论。
课堂产物三个可复用 prompt 模板、一个 AI 执行任务前检查表、一个科研任务拆解案例。
验收能把一个模糊科研需求改写成带材料、约束和验收标准的任务单。
D2 AM
2 小时

工具实操：Claude Code / WorkBuddy / Codex / GitHub

让老师知道工具怎么跑、怎么选、怎么避免把电脑搞乱；WorkBuddy 是教师友好的主力入口，Codex 是工程化对照。

  1. Claude Code + DeepSeek终端里的 Agent，能读文件、改文件、跑命令；只讲必要配置和安全边界。
  2. WorkBuddy教师更友好的主力工具，重点讲 Ask / Plan / Craft 或对应工作模式。
  3. Codex作为 WorkBuddy 的工程化平替，用来对比更接近程序员工具的 Agent 工作方式。
  4. Git / GitHub / Pages / VPN版本检查点、成果展示、网络路径和合规风险，只讲够用部分。
课堂产物完成一次工具可用性检查，并用 WorkBuddy 或 Claude Code 跑一次“先计划、再执行”的真实任务。
验收能说清 WorkBuddy、Claude Code、Codex 的分工，并知道 GitHub Pages 只是可选展示方式。
D2 PM
2 小时

WorkBuddy 科研效率工作流

把工具变成老师自己的科研流程，而不是停留在“会打开软件”。

  1. WorkBuddy 主力场景文献阅读与综述、论文写作、实验方案、数据整理、课程资料、项目申请。
  2. 工具分工WorkBuddy 适合科研任务和流程编排；Claude Code / Codex 适合代码项目和文件级修改。
  3. 个人科研 AI 工作流输入材料、工具处理、AI 输出、人工检查、保存结果、复用方式。
  4. 结课工作坊每位老师画出自己的工作流，并形成安全与验收清单。
课堂产物个人科研 AI 工作流图、工具选择表、三个个人 prompt 模板、上下文材料清单、安全与验收 checklist。
验收能用自己的真实科研任务说明：输入是什么、交给哪个工具、哪里人工检查、结果怎么保存和复用。

结课交付物

  • 个人科研 AI 工作流图：画出输入材料、处理工具、AI 输出、人工检查、保存位置和复用方式。
  • 工具选择表：列出每个科研任务优先用 WorkBuddy、Claude Code、Codex 还是其他工具，并写清理由。
  • 三个个人 prompt 模板：至少覆盖文献、写作、数据/课程/项目申请中的三个场景。
  • 上下文材料清单：列出每类任务要提前准备的论文、数据、要求、示例、格式和禁区。
  • 安全与验收 checklist：检查 API Key、隐私材料、事实来源、人工复核、版本保存和可复用记录。
  • 可选 GitHub Pages 展示页：如果需要公开展示工作流或课程资源，再把结果做成静态网页。

核心术语

  • LLM：大语言模型。它根据上下文预测下一个 token，不是一本不会错的百科全书。
  • Token：模型眼里的文字单位。中文一个字、英文单词片段、标点都可能算 token。费用和上下文窗口都和它有关。
  • 上下文窗口：模型一次能看到的材料范围。超过范围后，旧内容会被压缩或遗忘。
  • 幻觉：模型给出看似合理但没有证据的内容。解决方法不是“相信更贵模型”，而是给来源、跑验证。
  • API：软件之间约定好的接口。像餐厅点餐口：你按格式点菜，后厨按规则返回结果。
  • API Key：调用 API 的钥匙。它等于能花你额度的门禁卡，不能截图、不能提交到 GitHub。
  • Prompt：给模型的任务单，不是咒语。好 prompt 说清目标、材料、限制、输出形式、验收标准。
  • Coding Plan：AI 动手前的设计图。像装修前先看图纸，避免边拆边想。
  • Context Engineering：上下文工程。不是写一句神奇提示词，而是把模型需要看的文件、规则、例子、错误信息组织好。
  • Harness：把模型包起来的工作装置：指令、工具、文件、权限、测试、日志、人工确认。像实验台，而不只是口头命令。
  • Loop：Agent 的循环：计划 -> 修改 -> 运行 -> 观察 -> 修复 -> 记录。循环质量决定任务能不能稳定完成。
  • Agent：会使用工具并持续观察结果的 AI 助手。聊天模型只说，Agent 能读文件、改文件、跑命令。
  • Server / 服务器：远处一直开着的电脑。你打开网页、调用 API、访问模型，本质上常常是在和远处服务器说话。
  • 中转站 / Proxy / Relay：请求先到一个驿站，再被转给真正服务。它可能负责换路线、换格式、统一入口或隐藏复杂配置。
  • RAG：检索增强生成。先从资料库找相关材料，再让模型基于材料回答。适合问自己的文档。
  • MCP：模型上下文协议。让 AI 工具以统一方式连接外部工具和数据源。初学者先知道概念即可。
  • CLI：命令行界面。用文字发指令。AI 编程工具喜欢 CLI，因为命令可复制、可记录、可自动执行。
  • GUI：图形界面。用按钮、菜单、窗口操作。对人友好，但不如 CLI 容易被 Agent 稳定复现。

Claude Code + DeepSeek 配置

Windows PowerShell
$env:ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
$env:ANTHROPIC_AUTH_TOKEN="<你的 DeepSeek API Key>"
$env:ANTHROPIC_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-flash"
$env:CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-flash"
$env:CLAUDE_CODE_EFFORT_LEVEL="max"
claude --version
claude
macOS / Linux
export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_AUTH_TOKEN="<你的 DeepSeek API Key>"
export ANTHROPIC_MODEL='deepseek-v4-pro[1m]'
export ANTHROPIC_DEFAULT_OPUS_MODEL='deepseek-v4-pro[1m]'
export ANTHROPIC_DEFAULT_SONNET_MODEL='deepseek-v4-pro[1m]'
export ANTHROPIC_DEFAULT_HAIKU_MODEL=deepseek-v4-flash
export CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash
export CLAUDE_CODE_EFFORT_LEVEL=max
claude --version
claude
