课程安排

标准 8 小时，拆成 2 天半天

Day 1 先建立 AI 认知和指挥方法，Day 2 再做工具实操和 WorkBuddy 科研工作流。这样比把安装、概念和项目硬塞进 6 小时更稳。

D1 AM 2 小时

AI 认知与基础概念

让计算机小白先建立地图，不急着装工具，先知道 AI、API、服务器、中转站和终端这些词到底在说什么。

AI 发展史专家系统、搜索引擎、深度学习、ChatGPT、Claude、DeepSeek、Agent 工具。
LLM 基础模型、token、上下文窗口、幻觉、推理模型、费用、速率限制。
通俗概念API、API Key、Prompt、Coding Plan、服务器、中转站、CLI/GUI。
AI 工具地图聊天助手、搜索增强、科研阅读、WorkBuddy 类工作台、编程 Agent、自动化工具。

课堂产物

一张“AI 工具地图”和一张“我能听懂的术语表”。

验收方式

能用生活化语言解释 API、Prompt、Coding Plan、服务器、中转站、CLI/GUI 和幻觉。

D1 PM 2 小时

AI 指挥方法

先学会怎么把任务讲清楚，再去使用具体工具；让老师从“问一句”升级到“设计一套可检查的任务”。

Prompt Engineering目标、材料、限制、输出格式、验收标准。
Context Engineering把资料、文件、示例、错误信息摆上桌，让模型少猜。
Coding Plan / Harness / Loop先计划，再执行；用检查表、权限、日志和人工确认把 AI 放进工作装置。
科研任务拆解文献综述、论文润色、数据分析、课程资料整理、实验方案讨论。

课堂产物

三个可复用 prompt 模板、一个 AI 执行任务前检查表、一个科研任务拆解案例。

验收方式

能把一个模糊科研需求改写成带材料、约束和验收标准的任务单。

D2 AM 2 小时

工具实操：Claude Code / WorkBuddy / Codex / GitHub

让老师知道工具怎么跑、怎么选、怎么避免把电脑搞乱；WorkBuddy 是教师友好的主力入口，Codex 是工程化对照。

Claude Code + DeepSeek终端里的 Agent，能读文件、改文件、跑命令；只讲必要配置和安全边界。
WorkBuddy教师更友好的主力工具，重点讲 Ask / Plan / Craft 或对应工作模式。
Codex作为 WorkBuddy 的工程化平替，用来对比更接近程序员工具的 Agent 工作方式。
Git / GitHub / Pages / VPN版本检查点、成果展示、网络路径和合规风险，只讲够用部分。

课堂产物

完成一次工具可用性检查，并用 WorkBuddy 或 Claude Code 跑一次“先计划、再执行”的真实任务。

验收方式

能说清 WorkBuddy、Claude Code、Codex 的分工，并知道 GitHub Pages 只是可选展示方式。

D2 PM 2 小时

WorkBuddy 科研效率工作流

把工具变成老师自己的科研流程，而不是停留在“会打开软件”。

WorkBuddy 主力场景文献阅读与综述、论文写作、实验方案、数据整理、课程资料、项目申请。
工具分工WorkBuddy 适合科研任务和流程编排；Claude Code / Codex 适合代码项目和文件级修改。
个人科研 AI 工作流输入材料、工具处理、AI 输出、人工检查、保存结果、复用方式。
结课工作坊每位老师画出自己的工作流，并形成安全与验收清单。

课堂产物

个人科研 AI 工作流图、工具选择表、三个个人 prompt 模板、上下文材料清单、安全与验收 checklist。

验收方式

能用自己的真实科研任务说明：输入是什么、交给哪个工具、哪里人工检查、结果怎么保存和复用。

可选加课

什么时候值得加到 10-12 小时

不要为了显得完整而加课。只有当安装排雷、WorkBuddy 深水区或成果发布确实会阻塞学习时再加。

加 2 小时：安装排雷诊所

适合老师设备差异大、网络环境复杂时使用。专门处理 Node、Git、PowerShell 权限、代理、API 余额等问题。

加 2 小时：WorkBuddy 深水区

围绕文献综述、论文写作、数据整理和课程材料做更多真实案例。

加 2 小时：成果发布与 GitHub Pages

如果希望把工作流说明、课程资料或个人主页发布出来，再扩展 GitHub Pages 与静态网页。

课前准备

把课堂时间留给理解和练习

注册账号和下载安装最容易吞掉课堂时间，尽量提前一天完成。
