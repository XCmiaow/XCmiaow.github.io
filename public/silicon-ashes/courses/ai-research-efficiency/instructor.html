讲师手册

这门课的关键不是炫技，是降低恐惧

计算机小白最怕“我是不是做错了”。讲师要把每个黑箱拆成可观察的步骤，让老师知道哪里能试、哪里不能乱碰。

讲法原则

先给直觉，再给术语

每个新概念都按“生活类比 -> 软件例子 -> 最小操作 -> 常见错误”讲。

少讲定义

比如 API 先讲“点餐口”，再讲软件接口。

慢讲 Git

Git 是小白最大坡度。只讲检查点，不讲复杂分支模型。

不现场硬刚安装

个别电脑卡住时先用备用演示机，课后再排雷。

每 20 分钟给成果

能解释一个概念、能跑一次计划、能完成一张工作流卡片，信心才会起来。

常见卡点

  • PowerShell 不认识 npm：Node.js 没装好或终端没重启。
  • claude 无法调用：环境变量只在当前窗口生效，或 API Key/余额有问题。
  • 可选 GitHub Pages 404：仓库名、入口文件、Pages 分支、等待时间四选一。
  • AI 乱改文件：任务太大、没有计划、没有 Git 检查点。
  • WorkBuddy 输出太泛：缺输入材料、缺角色边界、缺人工验收标准。

课堂节奏

  • 每节课开头 5 分钟复盘上一节成果。
  • 新概念最多连续讲 15 分钟，随后给操作或例子。
  • 演示时只保留一个终端窗口和一个浏览器窗口。
  • 下课前必须让学员保存成果，不把半成品留在临时状态。
讲师话术

几句能稳定课堂的话

这些话不是口号，是给小白建立安全边界。

“终端不是让你背命令，是让 AI 和电脑有一张可复制的任务单。”
“Git 不是程序员仪式感，它是撤回按钮的高级版。”
“API Key 不是密码但比密码更容易花钱，所以按密码对待。”
“WorkBuddy 不是神奇按钮，它是工作台；材料摆得越清楚，产出越像样。”
“会提问不等于会写 prompt；会给材料、会验收，才叫会用 AI。”
