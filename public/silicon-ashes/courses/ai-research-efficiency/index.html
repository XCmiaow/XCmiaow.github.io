帮老师建立自己的科研 AI 工作流

四个半天，从听懂 AI 到设计工作流

Day 1 建认知和方法，Day 2 做工具实操和 WorkBuddy 科研效率工坊。每个半天都要产出可复用材料。

D1 AM
2 小时

AI 认知与基础概念

让计算机小白先建立地图，不急着装工具，先知道 AI、API、服务器、中转站和终端这些词到底在说什么。

  1. AI 发展史专家系统、搜索引擎、深度学习、ChatGPT、Claude、DeepSeek、Agent 工具。
  2. LLM 基础模型、token、上下文窗口、幻觉、推理模型、费用、速率限制。
  3. 通俗概念API、API Key、Prompt、Coding Plan、服务器、中转站、CLI/GUI。
  4. AI 工具地图聊天助手、搜索增强、科研阅读、WorkBuddy 类工作台、编程 Agent、自动化工具。
课堂产物一张“AI 工具地图”和一张“我能听懂的术语表”。
验收能用生活化语言解释 API、Prompt、Coding Plan、服务器、中转站、CLI/GUI 和幻觉。
D1 PM
2 小时

AI 指挥方法

先学会怎么把任务讲清楚，再去使用具体工具；让老师从“问一句”升级到“设计一套可检查的任务”。

  1. Prompt Engineering目标、材料、限制、输出格式、验收标准。
  2. Context Engineering把资料、文件、示例、错误信息摆上桌，让模型少猜。
  3. Coding Plan / Harness / Loop先计划，再执行；用检查表、权限、日志和人工确认把 AI 放进工作装置。
  4. 科研任务拆解文献综述、论文润色、数据分析、课程资料整理、实验方案讨论。
课堂产物三个可复用 prompt 模板、一个 AI 执行任务前检查表、一个科研任务拆解案例。
验收能把一个模糊科研需求改写成带材料、约束和验收标准的任务单。
D2 AM
2 小时

工具实操：Claude Code / WorkBuddy / Codex / GitHub

让老师知道工具怎么跑、怎么选、怎么避免把电脑搞乱；WorkBuddy 是教师友好的主力入口，Codex 是工程化对照。

  1. Claude Code + DeepSeek终端里的 Agent，能读文件、改文件、跑命令；只讲必要配置和安全边界。
  2. WorkBuddy教师更友好的主力工具，重点讲 Ask / Plan / Craft 或对应工作模式。
  3. Codex作为 WorkBuddy 的工程化平替，用来对比更接近程序员工具的 Agent 工作方式。
  4. Git / GitHub / Pages / VPN版本检查点、成果展示、网络路径和合规风险，只讲够用部分。
课堂产物完成一次工具可用性检查，并用 WorkBuddy 或 Claude Code 跑一次“先计划、再执行”的真实任务。
验收能说清 WorkBuddy、Claude Code、Codex 的分工，并知道 GitHub Pages 只是可选展示方式。
D2 PM
2 小时

WorkBuddy 科研效率工作流

把工具变成老师自己的科研流程，而不是停留在“会打开软件”。

  1. WorkBuddy 主力场景文献阅读与综述、论文写作、实验方案、数据整理、课程资料、项目申请。
  2. 工具分工WorkBuddy 适合科研任务和流程编排；Claude Code / Codex 适合代码项目和文件级修改。
  3. 个人科研 AI 工作流输入材料、工具处理、AI 输出、人工检查、保存结果、复用方式。
  4. 结课工作坊每位老师画出自己的工作流，并形成安全与验收清单。
课堂产物个人科研 AI 工作流图、工具选择表、三个个人 prompt 模板、上下文材料清单、安全与验收 checklist。
验收能用自己的真实科研任务说明：输入是什么、交给哪个工具、哪里人工检查、结果怎么保存和复用。
叙事调整

从“做网页项目”改成“科研工作流”

GitHub Pages 仍然保留，但只作为可选展示方式。课程主角改为 WorkBuddy 科研效率工作流，Codex 作为工程化平替和对照。

课程定位

老师不需要理解所有代码细节，但要理解三件事：AI 看到什么、AI 用什么工具处理、人在哪里检查和保存结果。

核心心智模型

  • LLM 不是资料库，是会根据上下文预测答案的生成器。
  • Prompt 是任务单，Context 是摆在桌上的材料。
  • Coding Plan 是设计图，Harness 是工作台，Loop 是迭代节奏。
  • WorkBuddy 做科研流程，Claude Code / Codex 做更工程化的文件和代码任务。
工具地图

先分清工具类型，再决定教学顺序

课堂上不要把所有 AI 工具讲成一锅粥。先分工，老师才知道什么时候该打开哪个工具。

类型代表工具适合任务
聊天助手ChatGPT / Claude / DeepSeek / Kimi解释概念、改写文字、头脑风暴。不要直接当事实来源。
搜索增强Perplexity / AI 搜索 / 浏览器插件找资料、追溯网页来源。重点看来源，不只看总结。
科研阅读NotebookLM / Elicit / Zotero 插件围绕论文集合提问、做文献矩阵、整理研究脉络。
科研工作台WorkBuddy把文献、写作、数据、课程材料、流程编排放在一个更适合教师的工作界面里。
编程 AgentClaude Code / Codex / Cursor / Windsurf读项目、改文件、跑命令、修 bug。适合作为 WorkBuddy 的工程化对照或平替。
自动化工具Zapier / Make / n8n / MCP / 自定义脚本把重复流程串起来。适合稳定流程，不适合还没想清楚的任务。
工作流选题

最终任务完全开放，但都落到科研 AI 工作流

老师可以选文献、写作、数据、课程资料或项目申请；验收统一看工作流是否讲清输入、工具、输出、人工检查和复用方式。

文献综述工作流

输入论文、检索式和研究问题，输出文献矩阵、主题脉络、争议点和下一步阅读清单。

论文写作工作流

输入提纲、实验结果和目标期刊风格，输出段落草稿、审稿人视角问题和修改清单。

数据整理工作流

输入表格、字段说明和目标图表，输出清洗建议、可视化草案和异常值检查表。

课程资料工作流

输入教学目标和已有讲义，输出课程大纲、案例、课堂练习和作业说明。

项目申请工作流

输入指南、研究基础和初步想法，输出结构化提纲、创新点打磨和风险清单。

最终交付

每位老师带走一套科研 AI 工作流包

这比“会打开某个工具”更有价值，因为它能在下次科研任务里复用。

个人科研 AI 工作流图

画出输入材料、处理工具、AI 输出、人工检查、保存位置和复用方式。

工具选择表

列出每个科研任务优先用 WorkBuddy、Claude Code、Codex 还是其他工具，并写清理由。

三个个人 prompt 模板

至少覆盖文献、写作、数据/课程/项目申请中的三个场景。

上下文材料清单

列出每类任务要提前准备的论文、数据、要求、示例、格式和禁区。

安全与验收 checklist

检查 API Key、隐私材料、事实来源、人工复核、版本保存和可复用记录。

可选 GitHub Pages 展示页

如果需要公开展示工作流或课程资源，再把结果做成静态网页。
