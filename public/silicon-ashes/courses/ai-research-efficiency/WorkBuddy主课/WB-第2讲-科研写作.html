第 2 讲 · 科研写作 · 文献管理 · 知识沉淀

论文翻译/润色/格式/投稿四件套 · Zotero 联动 · 封装你的第一个科研 Skill

时长
120 分钟
形式
讲授 + 实操 + 互评
产出
每人一个自定义 Skill

本讲目录

  1. 论文辅助四件套（上） — 单篇摘要 + 批量对比（25 min）
  2. 论文辅助四件套（下） — 翻译润色 + 审稿回复（30 min）
  3. Zotero 联动 — 批量文献分类（20 min）
  4. 封装第一个科研 Skill — 知识沉淀（25 min）
  5. 圆桌互评 — Skill 展示与优化（20 min）

1. 论文辅助四件套（上）

"AI 是你的第一轮文献筛选机"
你不需要逐篇精读 50 篇论文。AI 帮你做第一轮筛选——10 分钟把 50 篇变成 5 篇"必读"，然后你把省下的时间用在深度思考那 5 篇最重要的论文上。

单篇论文摘要

给 WorkBuddy 一篇论文（PDF），让它生成结构化摘要。

Prompt 模板（可直接使用）： 
请按照以下结构总结这篇论文：

1. 研究问题：作者试图解决什么问题？
2. 方法：用了什么方法/技术/催化剂？
3. 关键发现：最重要的 3 个实验结果是什么？
4. 创新点：与已有研究相比，最大的创新在哪里？
5. 局限与展望：作者提到了哪些不足和后续方向？
6. 与我的研究关联：[你是做什么方向的]——这篇论文对我的启示是什么？

请用中文回答，每个部分 2-4 句话。如果论文中某些信息缺失，请明确说明"论文未提及"。

批量文献比较

当你需要对比 5-10 篇论文之间的共同发现和矛盾点时：

Prompt 模板： 
我上传了 N 篇关于 [研究主题] 的论文。请帮我做以下分析：

1. 共同发现：这些论文的结论中，观点一致的 3-5 个核心发现是什么？
2. 矛盾点：哪些结论之间存在矛盾？（指出具体哪两篇论文在什么点上不一致）
3. 研究空白：基于这些论文的讨论部分，当前领域还有哪些未解决的问题？
4. 趋势判断：从这些论文看，该领域的研究方向正在如何演变？

输出格式：每个部分用小标题 + 要点列表。
破除误区："AI 读论文 = 偷懒，不认真做学问"
正解：AI 做第一轮筛选，省下时间深度思考最重要的论文。 一个真正优秀的科研工作者，不是花更多时间在机械阅读上，而是把时间用在最有价值的深度理解和批判性思考上。

2. 论文辅助四件套（下）

① 论文翻译：中文 ACS/Nature 风格英文

Prompt 模板： 
请将以下中文科研文本翻译成英文。

要求：
- 目标期刊风格：[ACS Catalysis / Nature Communications / Chemical Engineering Journal]
- 学术正式语气，避免口语化
- 保持专业术语的一致性（如"催化活性"始终用"catalytic activity"）
- [可选] 如果原文过长，适当拆分长句使其更符合英文表达习惯

原文：
[粘贴中文内容]

② 论文润色：提升学术语感和逻辑流

这不是简单的语法检查，而是提升学术语气、段落逻辑流、术语精确度和表达地道性。

Prompt 模板： 
请润色以下英文学术文本。

要求：
- 增强学术语气（更正式、更严谨）
- 优化段落之间的逻辑过渡
- 统一术语表达
- 保持原意不变，不要添加新内容
- 如有歧义，标注 [需确认] 让我判断

原文：
[粘贴英文内容]

③ 参考文献格式化

把散乱的参考文献信息整理成目标期刊的引用格式。

操作方式：给 WorkBuddy 一段参考列表（或 PDF 中的引用页），指定目标格式（如 ACS style, Vancouver, APA），AI 自动格式化输出。

④ 审稿回复策略

Prompt 模板： 
以下是审稿人对我的论文提出的意见。请帮我：

1. 逐条分析审稿人的核心关切是什么
2. 为每条意见建议回复策略（接受修改/解释说明/补充实验/礼貌反驳）
3. 帮我起草回复信的第一段和最后一段（表达对审稿人的感谢和尊重）

审稿意见：
[粘贴审稿意见]
关键警告：AI 处理过的每一句话都要人工验证
AI 在学术写作中最容易出两个问题：
① 术语漂移：AI 可能把你说的"催化活性位点"翻译成"catalytic site"但原文其实特指"Brønsted acid site"——细微偏差导致结论失真。
② 语义漂移：AI 在"润色"时可能微妙改变了你的原意，比如把"significantly improved"改成"dramatically enhanced"——在学术论文中，"显著"和"剧烈"是两种不同的量级。
验证原则：每一句 AI 修改过的内容，你都要逐句对照原文，确认意思是否一致。

3. Zotero 联动

"200+ 篇论文的图书馆里找路——AI 就是你的导航"
你已经在 Zotero 中积累了数百篇文献，但分类标签混乱、想找"用 DFT 计算催化机理"的论文需要手动翻半天。让 WorkBuddy 连接 Zotero，自动帮你归类、打标签、生成综述骨架。

典型场景

你的 Zotero 里有 200+ 篇催化领域论文，现在需要按方法论（实验/理论计算/机器学习）、理论框架（DFT/KMC/微动力学）、实证类型（模型催化/真实工况）三个维度重新分类。

操作流程

1
获取 Zotero API Key。登录 Zotero 网页版 Settings Feeds/API Create New Private Key 勾选"Allow library access" 复制生成的 Key。
2
WorkBuddy 连接 Zotero。在 WorkBuddy 中配置 Zotero MCP 或直接给出指令："请通过 Zotero API 访问我的文献库（User ID: [你的ID], API Key: [密钥]）"。
3
批量分类指令。"请分析我 Zotero 库中所有论文的标题和摘要，按以下维度分类：① 方法论（实验/理论计算/机器学习）② 催化体系（热催化/电催化/光催化）③ 核心反应类型。为每篇论文打上对应标签。"
4
生成综述骨架。"基于分类结果，生成一篇关于 [某研究方向] 的综述大纲，标注每部分对应的关键论文。"
演示环节：导师在自己的屏幕上演示完整的 Zotero 联动流程。建议学员提前准备好自己的 Zotero API Key。

4. 封装第一个科研 Skill

"Skill = 实验室 SOP 的数字化"
你实验室里一定有很多 SOP（标准操作流程）：怎么做 XRD 图谱分析、怎么写实验方案、怎么回复审稿人……把这些 SOP 封装成 Skill，以后一个指令就能触发整个流程。

什么是 Skill？

Skill 是 WorkBuddy 中的可复用的任务模板。它包含四个要素：

要素说明示例
名称Skill 的标识"论文摘要-ACS风格"
触发词哪些关键词会激活这个 Skill摘要、summarize paper、paper summary
工作流AI 执行的具体步骤提取标题分析摘要总结方法提炼创新格式化输出
输出规范结果呈现的格式要求Markdown 表格 + ACS 引用格式

操作步骤

1
描述需求。跟 WorkBuddy 说："我想创建一个 Skill，每次我给一篇论文，你自动生成 ACS 风格的结构化摘要，包含研究问题、方法、关键发现、创新点、与我研究的关联。"
2
AI 生成 Skill 模板。WorkBuddy 会自动生成 SKILL.md 文件和配套脚本。
3
保存为 Skill。将生成的文件保存到 WorkBuddy 的 Skills 目录下。
4
测试。给一篇论文说"帮我做摘要"，看 Skill 是否按照你的规范输出。
实操任务：每人从以下选项中选择一个创建 Skill：
① "论文摘要-ACS风格"（输入论文PDF，输出结构化摘要）
② "实验方案模板"（输入课题名+反应条件，输出标准化实验方案）
③ "审稿回复助手"（输入审稿意见，输出逐条回复策略+起草回信）
用时约 15 分钟，完成后导出 Skill 文件。
破除误区："Skill 建好就不能改了"
正解：好的 Skill 是用出来的，不断迭代。 第一次用发现模板不够好？改。用了一个月发现可以加新功能？加。Skill 就像实验室里的操作手册——写好了先用，用出问题再修订，最终形成你的"科研知识兵法"。

5. 圆桌互评

互评流程（每人 4 分钟）：
① 1 分钟展示 Skill：触发词是什么、解决什么问题、输出什么样
② 2 分钟其他人试用：用一段测试论文（或其他人的数据）测试 Skill
③ 1 分钟改进建议：其他人提出优化建议（模板是否清晰、输出是否规范、是否可扩展）

本讲小结

  • 论文辅助四件套：翻译（中英学术风格）、润色（学术语气+逻辑流）、格式（参考文献标准格式化）、投稿（审稿回复策略）
  • AI 写作的核心原则：每一句 AI 处理过的内容都需人工验证，警惕术语漂移和语义漂移
  • Zotero 联动：通过 API Key 连接，AI 可批量分类文献并生成综述骨架
  • 封装第一个 Skill：Skill = 数字化 SOP，建好使用迭代沉淀
