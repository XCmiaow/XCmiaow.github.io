1. Coding Plan = "装修前先画图"

B1 解决了"AI 在哪干活"的问题。现在 B2 解决另一个问题：你该怎么跟 AI 说话，才能让它干活干得漂亮？

答案的第一部分就是 Coding Plan——直译是"编码计划"，但对我们来说更好的翻译是"先出方案，确认了再动手"。

一个装修的比喻

你买的房子需要装修。你找到装修队，说："帮我把这 120 平米的房子装修好。"

如果装修队直接开始砸墙、铺地砖、改水电——你会疯掉的。你不知道他打算做什么风格、不知道预算会超多少、不知道会不会把承重墙砸了。你需要的是：先出设计图，你看过、修改过、确认了，再动工。

这就是 Coding Plan 的核心思想。你跟 AI 说"帮我写一篇文献综述"，AI 如果直接开始写 3000 字——你同样会疯掉。你不知道它用了什么框架、不知道覆盖了哪些文献、不知道重点放在哪。你需要的是：先出大纲和结构，你确认了方向，再让它写正文。

Plan vs Craft：两种沟通方式
不推荐（直接动手）："帮我写一篇关于 CO₂ 还原电催化剂的最新综述。"
AI 哗啦啦输出 5000 字，但结构不是你要的，文献不是最新的，方向偏了。重来的成本很高。

推荐（先 Plan 再执行）："先帮我规划一篇关于 CO₂ 还原电催化剂综述的结构和大纲，覆盖 2020-2025 年的进展，按催化剂类型分类，重点讨论选择性和稳定性。"
AI 先出一个结构：第一部分引言、第二部分均相催化、第三部分非均相催化、第四部分单原子催化……你看过 调整 确认 然后 AI 按你确认的框架写正文。

这和工作中的"汇报前先对提纲"一模一样

你在学术圈做了这么多年，对这个逻辑应该非常熟悉。你的博士生要写一篇论文，你不会让他写完了再给你看——你让他先写提纲、先搭框架。写完了再改，时间成本巨大；框架对了再填内容，效率翻倍。

AI 也是这样。它的"体力"近乎无限——写 5000 字还是 50000 字对它来说区别不大。但它的"方向感"需要你来校准。你用 Plan 模式给了它方向，它用 Craft 模式替你干活。这就是 WorkBuddy 里 Ask / Plan / Craft 三种模式的设计哲学：

模式类比什么时候用
Ask问同事一个简单问题"这个反应的条件是什么？""这组数据怎么解读？"——单次问答，不需要规划
Plan让同事先出方案，你审阅"帮我规划一个文献综述的结构""设计一个数据处理流程"——需要你的判断和调整
Craft同事确认方案后直接交付你已确认 Plan 说"按这个方案执行" AI 全自动出成果
核心收获：跟 AI 协作的最高效方式不是"一句话扔给它就完事"，而是"先盯方向 确认框架 再放手让它干"。这 2 分钟的 Plan 步骤，往往省掉 20 分钟的返工。

一个真实的例子

假设你要用 WorkBuddy 做一个"课题组介绍网页"。两种做法：

做法 A（没说 Plan）："帮我做一个课题组网页。"AI 用自己的审美和理解做出了一个页面——颜色不对、布局不是你想要的、信息不全。你说"不对，改这里、改那里、再加一段……"来回改了 8 轮，30 分钟过去了。

做法 B（先 Plan）："先帮我规划一个课题组网页的结构，包括：研究方向、成员介绍、论文发表、联系方式四个板块。用卡片式布局。"AI 给出一个结构方案。你说"成员那块加个头像框，研究方向要加图片。"AI 更新方案。你说"好的，按这个做。"AI 一次到位，5 分钟搞定。

同样的任务，不同的沟通方式，效率差 6 倍。这 6 倍的差距不是因为 AI 能力的差异——是因为你和 AI 的协作模式不同。

2. Workflow = "实验室 SOP 数字化"

如果说 Coding Plan 解决的是"单次任务怎么做"，那 Workflow 解决的就是"重复任务怎么做"。

什么是 Workflow？

Workflow 直译是"工作流"，本质上就是一套可重复的结构化任务流程。你做实验不会每次都重新想步骤——你有一套固定的 SOP（标准操作规程）：称量 混合 加热 冷却 检测 记录。Workflow 就是数字化的 SOP。

一个通用的 Workflow 模板如下：

通用任务 Workflow：
输入 AI 处理 输出 人工审阅 反馈 迭代

这个模板几乎适用于所有科研场景。我举一个具体的例子——每个老师都会遇到的事：

场景：每次新论文发布了，我要快速了解它
输入：把新论文 PDF 拖进 WorkBuddy
AI 处理：提取实验方法、标记创新点、总结结论、列出与我相关的地方
输出：一段 500 字的"一句话摘要 + 方法卡片 + 与我课题的联系"
人工审阅：你扫一眼确认关键信息没丢
反馈："这部分写得更详细点""补充一下催化剂的表征方法"
迭代：AI 更新，存入你的文献库

这个流程你今天就可以用。而且它不是一次性的——每次新论文来了，你都走这个流程。这就是 Workflow 的核心价值：把好的做法固定下来，让每一次都像第一次一样高效。

从"手动 Workflow"到"自动化 Workflow"

在过渡桥梁阶段，你用手动方式理解 Workflow 的概念。到了 WorkBuddy 主课的第 5 讲，你会把这种工作流封装为自动化的 WorkBuddy Workflow——比如设置"每天上午 9 点自动扫描最新文献、生成摘要、推送到你的飞书"。从"每次自己操作一遍"到"AI 自动帮你跑一圈"——这就是从 20 分钟的认知课到 10 小时实战课的跃迁路径。

一句话记住 Workflow："你做实验有一套 SOP，你用 AI 也有一套 SOP。"好的 Workflow 不会限制你的创造力——它把你从重复劳动中解放出来，让你有更多时间思考真正重要的东西。就像好的实验 SOP 不是限制你，而是让你不用每次重新想"先加 A 还是先加 B"。

3. API = "连接一切的万能胶水"

在 M2（LLM 基础知识）中，我们提到了 API 这个术语。在 B2 中，我们把它从"技术术语"转化为你真正能用的认知工具。

回到本质：API 到底是什么？

API = Application Programming Interface，直译是"应用程序编程接口"。但这个名字对非技术人员来说没有任何意义。我换一种说法：

API 就是程序之间的"酒店房卡"。你的手机上装了几十个 App——天气 App 知道明天几度、外卖 App 知道附近的餐厅、地图 App 知道实时路况。这些数据不是 App 自己"想"出来的——它们都是通过 API 从别的地方拿到的。

程序说话的方式
程序 A 想用程序 B 的数据或功能，它就拿着 API Key（房卡）去刷 B 的门。B 看一眼房卡："OK，你有权限，这是你要的温度数据 / 附近餐厅列表 / 实时路况。"

没有房卡 进不去。
房卡权限不够 只能进大厅，进不了 VIP 区。
房卡过期 需要重新申请。

API 不是技术细节——它是理解"这个世界上所有软件为什么能联动"的钥匙。

为什么你要理解 API？三个你马上能用的例子

例子一：Claude Code 为什么能用 DeepSeek 的脑子？

你在 WorkBuddy 里用的是 Claude 模型。但你也可以配置让它调用 DeepSeek 的 API——也就是说，界面是 WorkBuddy，脑子是 DeepSeek。这不是什么黑科技，就是 WorkBuddy 拿着你的 DeepSeek API Key 去刷 DeepSeek 的门，说"帮我处理这个请求"。理解了这个，你就理解了为什么你可以自由选择用哪个模型——以及为什么不同模型价格不同（因为不同的"酒店"收费标准不同）。

例子二：Zotero 为什么能被 WorkBuddy 控制？

在第 2 讲，你会学到 WorkBuddy 控制 Zotero——直接在你的文献库里搜索、添加、标注。这不是魔法，是 Zotero 暴露了一套 API，WorkBuddy 拿着"房卡"去刷——"帮我在第 23 号文件夹搜一下关于 MOF 催化剂的论文"——Zotero 返回结果。理解了 API，你对这些联动就不会再感到"不可思议"，而是"原来如此"。

例子三：你手机上的天气 App 为什么知道明天几度？

不是 App 开发者在每个城市放了个温度计。是 App 通过 API 从气象局的数据服务器拿的。每天、每小时、每次你打开 App，它都去"刷卡取数据"。这个例子虽然和科研无关，但它最好地说明了 API 的本质：程序之间的数据交换通道。

你在课程中的 API 学习路径：
M2：知道"API 是一个术语"（认知扫盲）
B2：理解"API 是程序之间的万能胶水"（概念深化）
M5：学会"API Key 安全管理"（安全实践）
第 2 讲：实操"Zotero API 控制"（实战验证）

这就是我们前面说的双环教学法——同一个概念在不同阶段以不同深度反复出现，直到你真正掌握。

从桥梁到主课：你准备好了

B1 + B2 总共 40 分钟，它们的作用不是让你成为程序员，而是让你从"旁观者"变成"指挥者"。你不需要会写代码，但你知道了：

  • AI 在哪干活——VSCode 的身体、终端的通道、Workspace 的边界
  • 怎么跟 AI 说话——Plan 模式先对齐方向，再放手执行
  • 为什么工具能联动——API 是程序之间的"酒店房卡"
  • 怎么把重复任务系统化——Workflow 是数字化的实验室 SOP

带着这四个认知进入 WorkBuddy 主课的 10 小时实战，你会发现：很多东西看起来是"AI 的神奇能力"，其实都是你在这 40 分钟里已经理解的底层逻辑。你不是在学 AI——你是在学会"指挥 AI 的方法论"。

B2 核心收获：跟 AI 协作的最高境界不是"它会什么"，而是"你知道它可以怎么为你工作"。Coding Plan 让你不会浪费 AI 的能力，Workflow 让你重复的事情不再重复，API 让你理解一切联动的本质。这三件事，就是你从"AI 用户"到"AI 指挥官"的通行证。
