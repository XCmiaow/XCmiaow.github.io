第一讲核心材料

基础概念：先把黑话翻译成人话

这部分不追求把神经网络讲透，而是让老师有一张能用的认知地图：AI 在哪里强、在哪里会胡说、为什么部署工具离不开 API 和终端。

AI 发展史

从规则到 Agent：一条非常短的长历史

这段历史不是为了背年份，而是为了理解今天的工具为什么长成这样。

1943
神经元数学模型

AI 的远祖不是聊天机器人，而是人们试图用数学描述“神经元怎么激活”。

1956
达特茅斯会议

人工智能作为研究领域正式出现。早期乐观，后来多次遇冷。

1960s
专家系统

把专家经验写成规则。像一本很厚的操作手册，能解决窄问题，但维护困难。

1997
Deep Blue

机器在国际象棋中战胜人类冠军，证明搜索和算力的威力。

2012
深度学习爆发

ImageNet 竞赛让神经网络重新成为主角。

2017
Transformer

现代大模型的关键架构出现，后来支撑了 GPT、Claude、DeepSeek 等模型。

2022
ChatGPT

AI 从研究圈进入大众视野，聊天成为入口。

2024-2026
Agent 工具普及

AI 开始不只回答问题，还会读文件、改文件、调用工具、参与完整工作流。

通俗概念

先把基础设施讲成人话

这组概念放在 Day 1 上午讲。老师听懂这些词，后面 Claude Code、WorkBuddy、Codex、API Key 和中转站才不会像魔法。

API

餐厅点餐口

你不用进厨房，只要按菜单格式点菜；软件不用知道后厨细节，只要按 API 规则请求结果。

API Key

能花额度的门禁卡

谁拿到这张卡，谁就可能调用服务并消耗你的额度，所以按密码对待。

Prompt

写给助理的任务单

不是“请帮我写好一点”，而是写清目标、材料、限制、输出格式和验收标准。

Coding Plan

装修前的设计图

先确认怎么改、改哪里、验收什么，再让 AI 动手，避免一边砸墙一边想。

Server / 服务器

远处一直开着的电脑

网页、模型和 API 多数不在你电脑里，而是在远处电脑上运行。

中转站 / Proxy / Relay

快递驿站

请求先到驿站，再转给真正服务；驿站可能换路线、换包装、统一入口。

CLI / GUI

文字指令台 / 按钮界面

GUI 像遥控器，CLI 像可复制的任务单；AI Agent 更容易稳定执行 CLI。

CLI、GUI，再回到 CLI

早期电脑只能敲命令，门槛高但可记录。后来 GUI 让普通人通过按钮使用电脑。今天 AI Agent 又把 CLI 推回中心，因为命令可以被复制、解释、重放和自动执行。

对老师来说，终端不是“学编程的惩罚”，而是给 AI 一个稳定操作台。

生活化类比

GUI 像遥控器，按钮清楚；CLI 像把完整指令写给助理，助理能照着做、能留下记录、也能把错误原样交回来。

术语表

AI 黑话翻译成人话

课堂上遇到术语先落到例子，不急着讲定义。

LLM

大语言模型。它根据上下文预测下一个 token，不是一本不会错的百科全书。

Token

模型眼里的文字单位。中文一个字、英文单词片段、标点都可能算 token。费用和上下文窗口都和它有关。

上下文窗口

模型一次能看到的材料范围。超过范围后，旧内容会被压缩或遗忘。

幻觉

模型给出看似合理但没有证据的内容。解决方法不是“相信更贵模型”，而是给来源、跑验证。

API

软件之间约定好的接口。像餐厅点餐口：你按格式点菜，后厨按规则返回结果。

API Key

调用 API 的钥匙。它等于能花你额度的门禁卡，不能截图、不能提交到 GitHub。

Prompt

给模型的任务单，不是咒语。好 prompt 说清目标、材料、限制、输出形式、验收标准。

Coding Plan

AI 动手前的设计图。像装修前先看图纸，避免边拆边想。

Context Engineering

上下文工程。不是写一句神奇提示词，而是把模型需要看的文件、规则、例子、错误信息组织好。

Harness

把模型包起来的工作装置：指令、工具、文件、权限、测试、日志、人工确认。像实验台，而不只是口头命令。

Loop

Agent 的循环：计划 -> 修改 -> 运行 -> 观察 -> 修复 -> 记录。循环质量决定任务能不能稳定完成。

Agent

会使用工具并持续观察结果的 AI 助手。聊天模型只说，Agent 能读文件、改文件、跑命令。

Server / 服务器

远处一直开着的电脑。你打开网页、调用 API、访问模型，本质上常常是在和远处服务器说话。

中转站 / Proxy / Relay

请求先到一个驿站，再被转给真正服务。它可能负责换路线、换格式、统一入口或隐藏复杂配置。

RAG

检索增强生成。先从资料库找相关材料，再让模型基于材料回答。适合问自己的文档。

MCP

模型上下文协议。让 AI 工具以统一方式连接外部工具和数据源。初学者先知道概念即可。

CLI

命令行界面。用文字发指令。AI 编程工具喜欢 CLI，因为命令可复制、可记录、可自动执行。

GUI

图形界面。用按钮、菜单、窗口操作。对人友好，但不如 CLI 容易被 Agent 稳定复现。

Prompt / Context / Harness / Loop

从一句话，到一套能复用的工作装置

提示词工程只是入口，上下文工程和 harness 才决定任务稳定性。

1

Prompt

写清楚任务：我要什么、给你什么、不要什么、结果长什么样。

2

Context

把资料、文件、例子、错误信息放进模型能看到的位置。

3

Harness

限制权限、指定工具、要求测试、保存日志，让 AI 在装置里工作。

4

Loop

计划、执行、观察、修复、提交。每一圈都要有证据。
